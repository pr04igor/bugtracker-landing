Bugtracker

Лендінг для бакалаврської роботи

Цей проєкт — це вебзастосунок для відстеження помилок, створений для допомоги командам у керуванні та контролі за програмними помилками. Користувачі можуть реєструватися, входити в систему, додавати баги та відстежувати їхній статус.

This project is a web-based bug tracking system designed to help teams manage and track software issues efficiently. It allows users to register, log in, add bugs, and track their status.

Про проєкт

Мета розробки

Метою проєкту є створення зручного вебзастосунку для ефективного відстеження програмних помилок і керування завданнями у командній розробці, щоб покращити керування завданнями та відстеження багів під час розробки.

Актуальність (теми)

У сучасному світі розробки програмного забезпечення своєчасне виявлення та усунення помилок є критично важливим для забезпечення якості продукту та задоволення користувачів. Зі збільшенням складності програмних систем зростає і потреба в ефективних інструментах для відстеження помилок. Розробка вебзастосунку Bugtracker є актуальною, оскільки він дозволяє командам розробників централізовано керувати інформацією про баги, оптимізувати процеси тестування та зменшити час на пошук і виправлення дефектів. Такий інструмент сприяє підвищенню продуктивності командної роботи та покращенню якості кінцевого продукту.

Основні завдання

  • Створити інтерфейс для керування завданнями
  • Забезпечити можливість додавання та відстеження помилок
  • Забезпечити багатокористувацький доступ
  • Реалізувати зміну статусу та коментування багів
  • Забезпечити фільтрацію та групування завдань
  • Впровадити базову адміністративну панель
  • Реалізувати базові механізми безпеки та резервного копіювання

Методологія дослідження

Використані технології

  • HTML5, CSS3 — для створення структури та стилізації вебсторінки
  • Адаптивна верстка — для забезпечення коректного відображення на різних пристроях
  • ARIA та семантична розмітка — для підвищення доступності сайту
  • Open Graph, Schema.org — для оптимізації SEO та соціального обміну
  • JavaScript — як основна мова для логіки клієнта
  • React — як основна бібліотека для побудови інтерфейсу користувача
  • Node.js + Express — для створення серверної частини застосунку
  • MongoDB — як база даних для зберігання користувацької інформації, багів та завдань
  • JWT (JSON Web Token) — для реалізації аутентифікації та авторизації
  • Jest — для модульного тестування окремих компонентів
  • Git + GitFlow — для контролю версій та структурованого процесу розробки

Використані середовища розробки (IDE)

  • Visual Studio Code — основне середовище розробки
  • GitHub — хостинг репозиторію та візуалізація командної роботи
  • Live Server — для локального перегляду змін у режимі реального часу
  • Chrome DevTools — для налагодження та тестування інтерфейсу

Очікувані результати

У результаті розробки очікується створення повнофункціонального вебзастосунку для відстеження помилок, який включає фронтенд та бекенд частини, реалізує роботу з базою даних, підтримує автентифікацію та багатокористувацький доступ, а також розробляється відповідно до GitFlow-стратегії з використанням системи контролю версій. Проєкт буде доступний як open source і придатний для подальшого розвитку або використання іншими командами.
